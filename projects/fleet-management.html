Vehicle Services Digital Platform

This project involved the development of a complete digital platform for a vehicle registration services company specialized in procedures such as vehicle registration, deregistration, ownership transfers, plate replacements, and compliance management.
Beyond creating a modern corporate website, the solution provides real-time tracking capabilities that allow both individual customers and companies managing large vehicle fleets to monitor the status of their procedures online. Through a secure integration with a centralized database, clients can instantly access updated information using their RFC, significantly improving transparency and customer experience.
To streamline internal operations, a custom desktop application was also developed using Python and Flet. By simply uploading an Excel file, company staff can automatically synchronize and update records stored in Supabase, eliminating manual data entry and ensuring accurate, real-time information across the platform.

Technology & Functionality

This project was developed as a comprehensive digital platform for a vehicle registration services company. The solution combines a public-facing website, a centralized cloud database, and an internal desktop application to streamline both customer interactions and operational workflows.

The web platform allows customers to access information about services such as vehicle registration, deregistration, ownership transfers, plate replacements, and compliance-related procedures. In addition to providing online visibility for the business, the platform offers a real-time tracking system that enables clients to check the status of their procedures directly through the website.

To support organizations managing large vehicle fleets, a dedicated consultation module was implemented. Customers can search and retrieve information associated with their RFC, allowing fleet managers to monitor multiple ongoing procedures efficiently without requiring manual assistance from company personnel.

On the operational side, a custom desktop application was developed using Python and Flet. The application allows employees to upload Excel spreadsheets containing updated procedure information. Once uploaded, the data is automatically validated, processed, and synchronized with the Supabase database, ensuring that customers always have access to accurate and up-to-date information.

By integrating web technologies, cloud databases, SQL, and process automation, the platform significantly improves customer transparency, reduces administrative workload, and provides a scalable solution capable of supporting both individual clients and enterprise fleet operators.

Architecture

A decoupled client-server architecture designed for high availability, secure data queries, and automated internal operations.

Conceptual illustration of the Architecture project on three principal elements.

Web Frontend (Hostinger): Built with native web technologies (HTML5, CSS3, Vanilla JS). It implements a frictionless Dual-Factor Query mechanism—requiring both the client's RFC and Vehicle Circulation Card—to securely expose real-time tracking data without traditional user authentication.

Database & Backend (Supabase): Acts as the system's Single Source of Truth utilizing PostgreSQL. All clients communicate via native REST APIs, eliminating middleware latency and streamlining CRUD operations.

Operational Desktop App (Python & Flet): A custom internal tool featuring an automated ETL pipeline. It parses administrative files locally (pandas for .csv, openpyxl for .xlsx) to validate data integrity before executing batch synchronizations to the cloud database.
