Technology & Functionality

The Fleet Management Platform was developed to simplify the administration and tracking of vehicle-related procedures for organizations managing large fleets.

The solution combines a public-facing web portal with a centralized cloud database, allowing fleet managers to monitor the status of active procedures through an RFC-based lookup system. This approach provides real-time visibility into ongoing requests while reducing the need for manual follow-up communication between clients and administrative personnel.

To support internal operations, the platform incorporates a workflow management system that stores procedure details, status updates, comments, and historical records in a centralized repository. This enables administrators to maintain accurate and up-to-date information throughout the lifecycle of each request.

A custom Python application was also developed to streamline large-scale data management. Through structured Excel imports, administrators can efficiently update multiple procedures at once, ensuring consistency across records while significantly reducing manual data entry.

By combining cloud-based data management, workflow automation, and self-service tracking capabilities, the platform improves operational efficiency, enhances transparency for fleet managers, and modernizes the administration of vehicle registration services.

Architecture

Láak Chatbot was designed using a modular architecture that separates the user interface, application services, language model orchestration, and organizational knowledge repository.

Each user request is enriched with relevant contextual information retrieved from the internal knowledge base before being processed by the selected language model. This approach improves response relevance while ensuring alignment with company-specific procedures and documentation.

By decoupling the frontend, backend, retrieval layer, and language models, the platform remains scalable, maintainable, and flexible enough to support future integrations and model experimentation.

Testing & Evaluation

To ensure response quality and reliability, the platform was evaluated using DeepEval, a specialized framework for testing large language model applications.


The evaluation process focused on answer relevance, factual consistency, contextual accuracy, and hallucination detection. These metrics provided objective insights into model behavior and enabled systematic comparison between different language models.


The evaluation process provided quantitative evidence of model performance, enabling informed decisions regarding model selection, response quality, and knowledge retrieval effectiveness.
