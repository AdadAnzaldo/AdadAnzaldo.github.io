Technology & Functionality

Láak Chatbot was developed as a locally hosted AI assistant designed to improve employee onboarding and knowledge accessibility within an organization.

The platform combines large language models with an internal knowledge repository, enabling the assistant to generate responses that are aligned with company-specific procedures, documentation, and organizational context. This approach allows employees to obtain relevant information quickly while maintaining consistency with internal processes.

A key feature of the platform is its support for multiple language models, including Ollama 3.2 and Qwen. Users can dynamically switch between models, enabling performance comparisons and adaptation to different use cases depending on response quality, speed, and contextual understanding.

By integrating retrieval-based knowledge enrichment with modern language models, Láak provides a practical solution for reducing onboarding time, improving information accessibility, and supporting knowledge sharing across teams.

Architecture

High-level architecture showing the interaction between the frontend, Flask API, LangChain orchestration layer, internal knowledge base, and models.

Láak Chatbot was designed using a modular architecture that separates the user interface, application services, language model orchestration, and organizational knowledge repository.

Each user request is enriched with relevant contextual information retrieved from the internal knowledge base before being processed by the selected language model. This approach improves response relevance while ensuring alignment with company-specific procedures and documentation.

By decoupling the frontend, backend, retrieval layer, and language models, the platform remains scalable, maintainable, and flexible enough to support future integrations and model experimentation.

Testing & Evaluation

To ensure response quality and reliability, the platform was evaluated using DeepEval, a specialized framework for testing large language model applications.


The evaluation process focused on answer relevance, factual consistency, contextual accuracy, and hallucination detection. These metrics provided objective insights into model behavior and enabled systematic comparison between different language models.


The evaluation process provided quantitative evidence of model performance, enabling informed decisions regarding model selection, response quality, and knowledge retrieval effectiveness.
